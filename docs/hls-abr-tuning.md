# HLS ABR Tuning

## Bitrate Ladder

The experimental HLS worker now creates three variants:

| Variant | Resolution | Video maxrate | Video bufsize | Audio |
| --- | ---: | ---: | ---: | ---: |
| 360p | 640x360 | 800k | 1600k | 96k |
| 720p | 1280x720 | 2200k | 4400k | 128k |
| 1080p | 1920x1080 | 4500k | 9000k | 160k |

The master playlist is generated by FFmpeg with:

```text
v:0,a:0,name:360p v:1,a:1,name:720p v:2,a:2,name:1080p
```

For silent input videos the audio mappings are omitted.

## Player Startup Strategy

Hls.js starts conservatively:

- save-data / 2G / 3G / mobile viewport: 360p
- desktop with strong reported downlink: 720p
- Safari uses native HLS and keeps browser-managed ABR

Starting at a lower variant generally feels faster because the first segments are smaller, so playback can begin before the player has enough bandwidth samples to safely climb the ladder.

## ABR Behavior

Hls.js estimates bandwidth from downloaded media fragments. It chooses the highest level that fits the estimate after applying safety factors. This implementation uses conservative factors to avoid repeated up/down switching:

- `abrBandWidthFactor: 0.85`
- `abrBandWidthUpFactor: 0.7`
- `abrEwmaFastVoD: 4`
- `abrEwmaSlowVoD: 15`

`capLevelToPlayerSize` prevents downloading 1080p for small rendered videos, which matters for reels and mobile grids.

## Buffer Strategy

Mobile buffers are intentionally smaller to reduce memory pressure during rapid scrolling:

- mobile max buffer: 12s target, 24s maximum
- desktop max buffer: 18s target, 36s maximum
- mobile back buffer: 10s
- desktop back buffer: 20s

## Segment Duration

Segments remain at 4 seconds. Shorter segments can react faster to network changes, but they increase manifest/HTTP overhead and can hurt R2/CDN efficiency. Four seconds is a practical VOD compromise for smoothness and cost.

## Critical Tests

- Chrome DevTools Slow 3G: starts at 360p, no stall before first frame.
- Chrome DevTools Fast 3G: starts low, climbs to 720p if stable.
- WiFi desktop: starts at 720p when reported network is strong, may climb to 1080p.
- Weak Android: verify memory stays stable while rapidly scrolling reels.
- iPhone Safari: native HLS still plays; Hls.js config is not used.
- Lightbox: opening/closing destroys Hls.js instance and preserves MP4 fallback.
- Network drop during playback: Hls.js should switch down rather than fatal fallback.
